2. 제품 개요

2.1 배경 및 목적

산업단지 내 제조 기업들은 장비 매뉴얼, SOP(표준작업절차서), 품질 보고서, 검사 기준서 등 방대한 문서 자산을 보유하고 있으나, 이를 체계적으로 검색·활용하기 어려운 상황이다. AX Flow는 문서 그래프 RAG(Retrieval-Augmented Generation) 및 온톨로지 기반 지식 그래프를 활용하여, 제조 현장 담당자가 자연어 질의만으로 필요한 정보를 신속·정확하게 찾을 수 있는 AI Transformation(AX) 시스템이다.

2.2 제품 비전

  • 제조 특화 AI 검색 — 장비·공정·원자재·품질 도메인에 최적화된 지식 그래프 기반 질의 응답
  • 문서 전처리 자동화 — 멀티 포맷(PDF, DOCX, HWP, XLSX) 파싱, OCR, 의미론적 청킹 파이프라인
  • 투명한 근거 제시 — 답변에 사용된 원본 문서·그래프 경로를 추적 가능한 형태로 시각화
  • 관리자 자율 운영 — 온톨로지 편집, 파라미터 튜닝, A/B 테스트를 GUI로 직접 수행

2.3 대상 사용자

4. 용어 정의

용어정의
GraphRAG지식 그래프(Knowledge Graph)와 RAG(검색 증강 생성)를 결합한 검색·답변 기법. 문서 내 엔티티 간 관계를 그래프로 구축하여 벡터 유사도 검색과 그래프 탐색을 동시에 수행한다.
온톨로지도메인 내 엔티티(장비, 공정, 원자재 등)와 관계(포함, 원인 등)의 유형을 정의하는 스키마. 그래프 추출의 기준 체계.
의미론적 청킹단순 길이가 아닌 문맥(Context) 단위로 텍스트를 분할하여 의미를 보존하는 전처리 방식.
엔티티지식 그래프의 노드. 장비명, 공정명, 원자재명 등 독립적 개체를 의미한다.
엣지지식 그래프에서 엔티티 간 관계를 나타내는 연결선. "원인", "포함", "참조" 등의 타입을 가진다.
MCPModel Context Protocol. AI 에이전트가 외부 도구(API, DB 등)에 접근하기 위한 표준 프로토콜.
RBACRole-Based Access Control. 사용자 역할(등급)에 따라 접근 가능한 문서·그래프 범위를 제어하는 보안 체계.
하이브리드 검색벡터 유사도(Vector Search)와 그래프 연결성(Graph Traversal)을 가중 결합하여 최적의 검색 결과를 도출하는 방식.
FaithfulnessRAG 성능 평가 지표. AI 답변이 참조 문서의 내용에 얼마나 충실한지를 측정한다.
Sub Agent특정 도메인·역할에 최적화된 프롬프트·파라미터를 가진 하위 AI 에이전트. 예: 품질전문가, 설비진단 전문가.

7. 채팅

7.1 채팅 영역

CHAT-01-01Sub Agent 셀렉트현재 활성(Active) 상태인 Sub Agent 목록을 드롭다운으로 노출한다. 사용자가 도메인·역할에 맞는 에이전트를 선택하여 대화를 시작한다.기본값: 선택 없음
CHAT-01-02대화 초기화 버튼현재 대화 컨텍스트를 초기화한다. 이전 대화 이력은 사이드바 히스토리에 보존된다.

설명가능 AI (XAI) 원칙

채팅 화면은 AI 답변의 근거를 투명하게 제시하여 사용자가 답변의 신뢰도를 직접 검증할 수 있도록 설계한다. 추론 경로(Graph Path)와 참조 청크 목록은 답변의 출처를 역추적할 수 있는 핵심 장치이다.
